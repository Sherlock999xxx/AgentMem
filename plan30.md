## 三、顶级 AI Agent 记忆平台对标

### 3.1 竞争格局

| 平台 | 核心定位 | 记忆模型 | 搜索能力 | 多Agent | 企业特性 |
|------|----------|----------|----------|---------|----------|
| **Mem0** | 开发者优先 | 分层记忆 | 向量+语义 | 基础 | 托管服务 |
| **Letta** | 持久化Agent | 心智模型 | 混合搜索 | 多Agent | API优先 |
| **Agno** | 多Agent编排 | Agentic Memory | 知识库 | 强大 | 生产导向 |
| **AutoGen** | 多Agent编排 | 会话历史 | 外部检索 | 强大 | 研究导向 |
| **Claude Memory** | 上下文管理 | 持久上下文 | 语义 | 多模态 | 云原生 |
| **AgentMem** | 生产级记忆 | 多层记忆 | 5种引擎 | 协作 | 全特性 |

### 3.2 Mem0 API 参考

```python
# Mem0 Python SDK 基本用法
from mem0 import MemoryClient

client = MemoryClient(api_key="your-api-key")

# 创建记忆
client.add(
    [{"role": "user", "content": "I love hiking on weekends"}],
    user_id="alice",
)

# 搜索记忆
client.search("What does Alice like to do?", user_id="alice")

# 列出所有记忆
client.get_all(user_id="alice")

# 更新记忆
client.update(memory_id="<id>", data="Alice loves mountain hiking")

# 删除记忆
client.delete(memory_id="<id>")
client.delete_all(user_id="alice")
```

// crates/agent-mem-core/src/lib.rs
pub enum CoreError {
    Storage(String),
    Database(String),
    Cache(String),
    NotFound(String),
    InvalidInput(String),
    // ...
}

impl std::error::Error for CoreError {}

// 各模块错误转换为 CoreError
impl From<StorageError> for CoreError {
    fn from(e: StorageError) -> Self {
        CoreError::Storage(e.to_string())
    }
}

pub enum QueryIntent {
    /// 精确记忆检索
    Recall { memory_id: String },
    /// 概念搜索
    Search { query: String },
    /// 学习新信息
    Learn { content: String },
    /// 忘记信息
    Forget { pattern: String },
    /// 总结
    Summarize { scope: Scope },
    /// 关系查询
    Relate { entity: String },
}

pub struct IntentClassifier {
    llm: Option<Arc<dyn LLM>>,
    rules: Vec<IntentRule>,
}

impl IntentClassifier {
    pub async fn classify(&self, query: &str) -> Result<QueryIntent> {
        // 1. 规则匹配
        if let Some(intent) = self.match_rules(query) {
            return Ok(intent);
        }

        // 2. LLM 分类
        if let Some(llm) = &self.llm {
            self.classify_with_llm(llm, query).await
        } else {
            Ok(QueryIntent::Search { query: query.to_string() })
        }
    }
}

pub struct AgenticMemoryConfig {

    pub enabled: bool,

}

/// Agentic Memory 管理器
pub struct AgenticMemoryManager {
    config: AgenticMemoryConfig,
    llm: Arc<dyn LLM>,
    memory: Arc<MemoryManager>,
}

impl AgenticMemoryManager {
    /// 从对话中提取记忆
    pub async fn extract_from_conversation(
        &self,
        messages: &[Message],
    ) -> Result<Vec<MemoryItem>> {
        let prompt = self.build_extraction_prompt(messages);
        let response = self.llm.generate(&prompt).await?;

        self.parse_extracted_memories(&response)
    }

    /// 自主决定是否存储记忆
    pub async fn should_remember(
        &self,
        memory: &MemoryItem,
    ) -> Result<bool> {
        let importance = self.memory.score_importance(memory).await?;
        Ok(importance >= self.config.importance_threshold)
    }

## 十一、关键决策记录

### 决策 1: 为什么创建新的 Crate 而非修改现有代码

**问题**: 为什么不直接在 agent-mem-core 中添加 VikingFS 功能?

**决策**: 创建独立的 `agent-mem-vikingfs` crate

**理由**:
1. **模块边界清晰**: VikingFS 是可选功能，不影响现有用户
2. **依赖隔离**: VikingFS 可能引入新依赖 (radix-trie)，不影响核心模块
3. **独立发布**: 可以单独版本控制和发布
4. **渐进采用**: 用户可以选择是否启用 VikingFS

### 决策 2: 为什么实现代码沙箱使用 WASM

**问题**: 为什么选择 WASM 而非其他沙箱技术?

**决策**: 使用 Wasmtime 作为沙箱运行时

**理由**:
1. **内存安全**: WASM 强制内存隔离
2. **性能**: 接近原生速度
3. **生态**: Rust 原生支持
4. **可移植**: 跨平台支持

**替代方案**:
- Docker 容器: 太重，启动慢
- gVisor: 需要额外依赖
- 纯 Rust 解释器: 开发和维护成本高

### 决策 3: 为什么意图分析使用混合方法

**问题**: 为什么同时使用规则和 LLM?

**决策**: 规则优先，LLM 降级

**理由**:
1. **速度**: 规则匹配 < 1ms
2. **成本**: LLM 调用成本高
3. **可靠性**: 规则可预测，LLM 可能不一致
4. **退化能力**: 无 LLM 时仍可工作
